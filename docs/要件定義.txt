1. プロジェクト概要
プロジェクト名: 自分だけのオリジナルスタートページ制作
目的:
煩雑なブックマークを整理し、目的のサイトへ素早くアクセスできるようにする。
視覚的に好みで、癒やしを感じられるデザインのスタートページを作成し、日々のインターネット利用のモチベーションを向上させる。
ターゲットユーザー: 製作者自身
利用シーン: PCブラウザ起動時、新規タブ表示時など、インターネットアクセスの起点として。

2.1.2. 時計表示機能 (モックアップにて実装済み)
デジタル形式 (HH:MM:SS) で現在時刻をリアルタイム表示。
ヘッダー中央部に大きく表示。
起動時のフェードインアニメーション。
2.1.3. 日付・曜日表示機能 (モックアップにて実装済み)
現在の日付と曜日を左カラムに表示。
2.1.4. 検索窓機能 (モックアップにて実装済み)
ヘッダー部、時計の下に配置。
Google検索に連携。Enterキーまたは検索ボタンで検索実行。
(検討事項) 検索エンジンの選択機能の追加。
2.1.5. ブックマーク管理機能 (モックアップにて基本CRUD実装済み)
中央カラムに配置。
カテゴリ管理:
ユーザーが任意の名前でカテゴリを動的に作成・編集・削除できる。
カテゴリ追加はモーダルウィンドウを使用。
カテゴリ編集はプロンプト、削除は確認ダイアログを使用。
ブックマーク管理:
各カテゴリ内にブックマーク (URL、タイトル) を登録・編集・削除できる。
ブックマーク追加・編集はモーダルウィンドウを使用。
ブックマーク削除は確認ダイアログを使用。
URLは http(s):// なしでも自動補完。
表示:
カテゴリごとにブックマークをリスト表示 (PCでは最大3カラム)。
ホバー時にカテゴリ編集・削除、ブックマーク追加ボタンを表示。
ホバー時にブックマーク編集・削除ボタンを表示。
(改善点) 各ブックマークにFaviconを表示する。
2.2. 追加機能 (初期リリース後、順次実装予定)
2.2.1. 天気予報表示機能 (モックアップではプレースホルダー)
左カラム、日付の下に配置。
Yahoo!天気予報API (または類似API) と連携。
表示項目: 天気アイコン、現在の天気、地域名、現在の気温、最高/最低気温。週間予報も検討。
2.2.2. ニュースフィード表示機能 (未実装)
右カラムまたは他の適切な場所に配置。
ユーザーが指定したRSSフィードを取得し、記事タイトルや概要を表示。
複数のRSSフィードを登録・管理できる。
2.2.3. ToDoリスト機能 (モックアップにてシンプル版実装済み)
右カラムに配置。
タスクの追加 (入力フィールド＋ボタン)、完了チェック、削除機能。
(将来拡張) リマインダー機能、タスクへのタグ付け、タグによる絞り込み・ソート。
2.2.4. メモ帳機能 (モックアップにてシンプル版実装済み)
右カラム、ToDoリストの下に配置。
簡単なテキストメモの入力、保存 (ボタンクリック)、表示。
(将来拡張) 複数メモ管理、タイトル付け、簡単な書式設定。
2.3. 将来的な拡張機能 (任意)
デザイン・レイアウトのカスタマイズオプション追加 (フォントファミリー、配色テーマ、要素の表示/非表示設定など)。
キーボードショートカットの拡充。
3. デザイン・レイアウト要件 (モックアップ mockup.html に基づく)
3.1. 基本レイアウト:
PC表示時 (幅768px以上): 3カラム構成 (grid-template-areas: 'left center right')
ヘッダー部 (画面上部、全幅): 時計、検索窓を中央揃えで配置。
左カラム (left-column): 日付・曜日、天気予報。
中央カラム (center-column): ブックマーク。
右カラム (right-column): ToDoリスト、メモ帳。
モバイル表示時 (幅768px未満): 1カラム構成 (grid-template-areas: 'left' 'center' 'right')
ヘッダー部は維持。各カラムが縦に積まれる。
3.2. デザインテーマ: 「自然や風景など、癒やされる感じ」
背景: Unsplashからの高画質な自然/風景画像が全画面背景。
UI要素のスタイル:
Tailwind CSSをベースとしたモダンでクリーンなデザイン。
情報表示エリアや入力エリアの背景には blur-bg (半透明の白色背景＋背景ぼかし) を使用し、ガラスモーフィズム風の質感を演出。
主要なコンテナは角丸 (rounded-xl など) を適用。
テキストカラーは主に白系 (背景画像とのコントラストを考慮) とグレー系。アクセントカラーは青系 (bg-blue-500 など)。
(検討事項) テーマに合わせてフォントファミリー (Google Fontsなどから選択)、及び配色の微調整。
アイコン: Font Awesomeを使用し、視認性と操作性を向上。
3.3. インタラクション:
時計、背景画像のフェードインアニメーション。
検索窓のフォーカス時にシャドウ効果。
カテゴリやブックマークのホバー時にアクションボタン (編集、削除、追加) が表示される。
カテゴリ追加、ブックマーク追加/編集はモーダルダイアログ (category-modal, bookmark-modal) を使用。
4. 非機能要件
4.1. 動作環境: PCの主要なモダンブラウザ (Chrome, Firefox, Edgeなど)。レスポンシブ対応によりスマートフォンでも基本的な閲覧は可能。
4.2. パフォーマンス: 快適な表示速度、スムーズな操作感。APIリクエストは適切に行う。
4.3. データ永続化: ブックマーク、ToDoリスト、メモなどのユーザーデータは、ブラウザのローカルストレージに保存する。
4.4. 保守性: 製作者自身がHTML, CSS (Tailwind CSS), JavaScriptで開発・メンテナンス可能であること。
4.5. セキュリティ:
* Unsplash APIキーについて: Access Keyはクライアントサイドでの使用が一般的だが、Secret Keyは原則クライアントサイドに含めない。現在のモックアップではSecret Keyも記載されているため、UnsplashのドキュメントでAccess Keyのみで動作するか再確認し、不要であれば削除、または必要な場合はサーバーサイドプロキシなどの対策を検討する。
5. 技術スタック (モックアップで使用された技術を反映)
フロントエンド: HTML5, CSS3, JavaScript (ES6+)
CSSフレームワーク/ユーティリティ: Tailwind CSS
アイコンライブラリ: Font Awesome
外部API:
Unsplash API (画像表示)
Yahoo!天気予報API (または類似API、天気予報表示用)
データ永続化: ブラウザ ローカルストレージ (localStorage)
